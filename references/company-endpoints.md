## ByteDance ✅

- **Portal**: https://jobs.bytedance.com
- **API**: `POST https://jobs.bytedance.com/api/v1/search/job/posts`
- **Body**: `{"keyword": "Agent", "limit": 50, "offset": 0, "portal_type": 1, "job_category_id_list": [], "location_code_list": [], "subject_id_list": []}`
- **Required headers**: `Origin: https://jobs.bytedance.com`, `Referer: https://jobs.bytedance.com/social`
- **Proxy**: ❌ No proxy (proxy returns 405 Method Not Allowed)
- **Fields**: `data.job_post_list[].title`, `city_info_list[0].cn_name`, `subject_name`
- **Pagination**: `data.total` controls page count, 50 per page
- **Known count**: ~73 L1 jobs (2026-04-10)

## Tencent ✅

- **Portal**: https://careers.tencent.com
- **API**: `GET https://careers.tencent.com/tencentcareer/api/post/Query?keyword={kw}&pageIndex={n}&pageSize=50&language=zh-cn&area=cn`
- **Proxy**: ❌ No proxy
- **Fields**: `Data.Posts[].RecruitPostName`, `LocationName`, `BGName`
- **Pagination**: `Data.Count` controls, 50/page
- **Known count**: ~139 L1 jobs (2026-04-10)

## Alibaba ✅ (Playwright DOM)

- **Actual portal**: `https://talent-holding.alibaba.com` (holding group, not talent.alibaba.com)
- **Direct API**: `POST https://talent.alibaba.com/position/search` → **❌ 403 (exit IP blocked by Alibaba risk control)**
- **Fallback**: Playwright rendering — fill search box `input[placeholder="输入关键词搜索职位"]` → Enter → DOM scrape `.job-card` list
- **Note**: URL keyword parameter doesn't work with SPA; must fill search box to trigger filter
- **Parse format A** (Alibaba holding): `Title\nUpdated DATE\nCategory-Type\nCity` (4 lines)
- **Known count**: ~11 L1 jobs (2026-04-10)

## Aliyun ✅ (Playwright DOM)

- **Portal**: https://careers.aliyun.com
- **Direct API**: `POST https://careers.aliyun.com/campus-recruitment/position/list` → **❌ 403**
- **Fallback**: Playwright rendering, same approach as Alibaba
- **Parse format B** (Aliyun): `Title\nUpdated DATE\nCity` (3 lines, no category)
- **Distinguishing A/B**: Regex `^[\u4e00-\u9fa5]+类[-—]` matches category line
- **Known count**: ~17 L1 jobs (2026-04-10)

## MiniMax ✅ (Feishu ATS + proxy, V2.0.0 New)

- **Parent company**: MiniMax (Hailuo AI)
- **Official site**: `https://minimaxi.com/careers` (Chinese, CSR, links to Feishu ATS)
- **Feishu ATS**: `https://vrfi1sk8a0.jobs.feishu.cn/379481/`
- **Feishu ATS project IDs**:
  - Experienced hire: `7496820276634634537`
  - Campus: `7495675705720965415`
  - Intern: `7352753013591755047`, `7572193079274801454`
- **Collection method (V2.0.0)**:
  - Direct connection → ❌ (IP blocked)
  - curl + proxy → ❌ (TCP connects, but `_signature` can't be forged)
  - Camoufox + proxy → ❌ (missing `libgtk-3.so.0`)
  - **✅ `playwright` Chromium + HTTP proxy**: intercept `/api/v1/search/job/posts` response
- **Actual API**:
  ```
  POST https://vrfi1sk8a0.jobs.feishu.cn/api/v1/search/job/posts
       ?keyword=<kw>&limit=10&offset=0&portal_entrance=1&_signature=<JS-generated>
  Response: { "data": { "job_post_list": [...], "count": <total> } }
  ```
- **Key limitation**: `_signature` is dynamically generated by browser JS, cannot be manually constructed
- **Known count**: ~47 L1 jobs (2026-04-10)

## Collection Strategy Priority

```
1. Direct HTTP JSON API (ByteDance/Tencent)           — Fastest, zero browser overhead
2. Playwright + proxy + request interception (Feishu ATS) — MiniMax, needs proxy, ~2-3min
3. Playwright DOM search box scraping (Alibaba/Zhipu/Kimi) — 5-10x slower, bypasses risk control
4. BOSS/Liepin                                        — Abandoned (requires login)
```

## Network Limitations

| Scenario | Status | Solution |
|----------|--------|----------|
| Tencent/ByteDance API direct | ✅ Normal | — |
| Alibaba IP risk control | ❌ 403 | Playwright headless browser |
| Feishu ATS exit IP blocked | ❌ | HTTP proxy (configure your own) + playwright |
| Proxy POST method rewrite | ❌ ByteDance 405 | ByteDance/Tencent don't use proxy |
| Old CLI CONNECT proxy (historical) | ❌ Not supported | Use standard playwright (current approach) |
| Camoufox + proxy | ❌ Missing libgtk-3 | Use system playwright (V2 approach) |
